### 技术栈

| 层 | 技术 |
|----|------|
| 前端构建 | Vite + TypeScript |
| 游戏渲染 | 纯 DOM 操作（innerHTML 模板字符串） |

## 二、文件使用方式

### 2.1 玩家使用（运行游戏）

#### 免安装版（推荐快速体验）
```
release/win-unpacked/锈甲长歌.exe
```
- 双击即可运行，无需任何安装
- 整个 `win-unpacked` 文件夹可拷贝至 U 盘或任意目录

#### 安装版
```
release/锈甲长歌 Setup 0.0.0.exe
```
- 双击运行安装向导
- 支持自定义安装路径
- 自动创建桌面快捷方式和开始菜单项
- 可通过 Windows「添加/删除程序」卸载

### 2.2 开发者使用

#### 环境要求
- Node.js ≥ 18
- npm ≥ 9

#### 安装依赖
```bash
cd game
npm install
```

#### 开发模式（推荐）
```bash
npm --prefix game run dev
```
- 启动 Vite 开发服务器 + Electron 窗口
- 修改 `src/` 下任意文件后自动热更新
- 也可在浏览器打开 `http://127.0.0.1:5173/` 测试（无需 Electron）

#### 仅启动 Web 端
```bash
npm --prefix game run dev:web
```
- 只启动 Vite 开发服务器，浏览器访问 `http://127.0.0.1:5173/`

#### 类型检查
```bash
npm --prefix game run typecheck
```
- 运行 `tsc --noEmit`，验证所有 TypeScript 类型

#### 运行测试
```bash
npm --prefix game run test:run
```
- 运行 Vitest，输出 8 个测试结果

#### 生产构建（Web）
```bash
npm --prefix game run build:web
```
- 先运行 typecheck，然后 vite build
- 产物输出到 `renderer/` 目录

#### 完整打包（Windows .exe）
```bash
npm --prefix game run build
```
- 先 build:web，再 electron-builder
- 产物输出到 `release/` 目录
- 耗时约 5-10 分钟

## 三、游戏基本操作

### 3.1 通用操作

| 操作 | 方式 |
|------|------|
| 交互/调查 | 点击场景中的 **圆形光点（热点）** |

### 3.2 各谜题专属操作

| 谜题 | 操作方式 |
|------|----------|

| 法阵摆放 | 为每个方位点击选择神兽 |

| 赶羊 | **按住小羊拖拽**到画面中央的绿色圆形靶心，倒计时归零时全部在圈内即胜 |

## 四、主线全流程攻略

### 第一关 · 书房的秘密

**剧情概要**：农家青年受领主举荐前往都城王宫任书记官，在书房整理文件时通过调查文书、账目和密信，揭开了国王发动战争的财政困境与外部勒索。

---

#### 场景 1：农舍（送别）

- 自动播放父母送别对话
- 对话结束后点击「启程」热点前往王宫

#### 场景 2：王宫门口

1. 点击左侧 **小丑** 对话，了解布告栏和国王书房的信息
2. 点击右侧 **布告栏** 阅读三份布告（税赋调整 / 异端清查 / 征兵书记官招募）
3. 阅读完毕后，点击中央 **守卫** 应聘书记官

#### 场景 3：谒见国王

- 自动播放国王任命对话
- 对话结束后点击左下「去二楼」

#### 场景 4：二楼走廊

- （可选）依次点击四幅 **画像**，查看历代国王自述
- 点击右侧「书房门」进入书房

#### 场景 5：书房调查（核心）

依次调查以下四个关键线索：

| 顺序 | 热点 | 位置 | 获得信息 |
|------|------|------|----------|
| ① | **征兵起草书** | 桌面中央 | 国王征召 16-60 岁男子，参军免三年赋税 |
| ② | **财政报告** | 桌面右侧 | 国库赤字 16080 英镑，连续两年入不敷出 |
| ③ | **搜寻桌底** | 桌板下方 | 先点击搜寻，再点击出现的密信 |
| ④ | **残局棋盘** | 桌面左侧 | 解开棋局获得兵力报告（仅 1870 正规军） |

> **棋局解法**：依次选择「白后逼近王侧」→「白象封住退路」→「白后完成将杀」

- （可选）点击书架上的 **历书** 查看四代国王变迁

全部调查完成后，点击右上角「面见国王」，自动跳转回谒见厅，国王准许你加入军队。场景中会出现「离开王宫」热点，点击前往教堂。

### 第三关 · 战争的代价

**剧情概要**：主角抵达前线军营，在生火、医帐登记、村庄救援和战场亲历中目睹战争的真实代价——补给匮乏、平民受害、宣传与现实的巨大落差。

---

#### 场景 7：寒夜军营

1. 点击 **木堆** 进入生火小游戏

#### 谜题 D：生火

保持 **火芯温度 38-76** 和 **鼓风力度 34-72** 在绿色区间内：

| 按钮 | 效果 |
|------|------|
| 🔥 敲击火石 | 温度 +18，风力 -8 |
| 🌿 添引火绒 | 温度 +10，风力 -4，进度 +7~18 |
| 💨 缓缓鼓风 | 风力 +16，温度 +2~7，进度 +4~14 |

> 提示：温度/风力都在绿色区间时，进度上涨最快。先用火石升温，再用鼓风调风力，最后反复添火绒。

2. 生火成功后，与 **骑士** 对话接受命令
3. 点击右下「去医帐」

#### 场景 8：医帐

4. 点击左侧 **家书** 进入匹配

按住左侧家书卡片拖到右侧对应遗体：

|----------|----------|
| 手上戴戒指，手臂带伤 | **家书一**（左肩受创，药很少） |
| 头发花白，旧伤累累 | **家书二**（年岁已老，求卸甲归田） |

6. 点击右下「列阵出发」，自动跳转

#### 谜题 G：赶羊

- 羊从牧场中心向四个方向快速跑开
- **按住并拖动小羊**，把它们拖回画面中央的**绿色圆形靶心**
- 羊被拖入圈内后会困住 1 秒（变为半透明绿色），随后重新向外跑

- 一旦有羊跑出画面边界，游戏立即失败

## 六、常见问题

### 运行问题

**Q：双击 exe 没反应？**
A：检查 `win-unpacked` 文件夹是否完整（约 200MB）。杀毒软件可能拦截，尝试添加信任。

**Q：窗口显示不全？**
A：游戏设计分辨率为 16:9，最低 1280×720，建议 1920×1080。开发模式按 F11 全屏。

**Q：如何卸载？**
A：安装版通过 Windows「设置 → 应用 → 锈甲长歌 → 卸载」。免安装版直接删除文件夹。

### 游戏问题

**Q：生火游戏进度条不动？**
A：温度和风力必须同时在绿色区间内，进度才会稳定上涨。单独操作一项很难成功。

**Q：推箱子卡住了？**
A：箱子只能推动一格，前方必须是空地且无其他箱子阻挡。如卡死，点击「重新搜路」重来。

**Q：赶羊太难？**
A：优先拖住跑得最快、最接近画面边缘的羊。拖回绿色圆形靶心后羊会困住 1 秒再跑。倒计时最后几秒要确保所有羊都在圈内。

**Q：对话结束后不知道去哪？**
A：观察场景中的发光圆形热点。线索簿会显示当前场景标题帮助你判断进度。

### 开发问题

**Q：如何新增场景？**
A：在 `narrativeData.ts` 中添加 `SceneDefinition`，在 `main.ts` 的 `handleAction` 中注册对应的 action。

**Q：如何新增谜题？**
A：在 `main.ts` 中定义新的 `OverlayState` 接口、编写 `render*Puzzle` 渲染函数、在 `handleCommand` 中添加对应的命令处理逻辑。

**Q：如何替换美术资源？**
A：将 PNG 文件放入 `public/art/` 对应关卡目录，修改 `narrativeData.ts` 中场景的 `background` 路径。缺失场景可在 `generatedSceneArt.ts` 中用 SVG 补画。
