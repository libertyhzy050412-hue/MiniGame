# 战争与和平 v1 开发计划

## 1. 产品定义

- 玩法形态以固定场景点击解谜为准，不按横版卷轴处理。
- 主要依据来自 [analysis/extracted/第一关.txt](analysis/extracted/第一关.txt)、[analysis/extracted/第二关.txt](analysis/extracted/第二关.txt)、[analysis/extracted/第三关.txt](analysis/extracted/第三关.txt)、[analysis/extracted/第四关.txt](analysis/extracted/第四关.txt) 与 [analysis/pdf_pages/page-03.png](analysis/pdf_pages/page-03.png)、[analysis/pdf_pages/page-07.png](analysis/pdf_pages/page-07.png)、[analysis/pdf_pages/page-08.png](analysis/pdf_pages/page-08.png)。
- 视觉方向以浅底色、扁平矢量、中世纪插画风为准，优先复用现有美术资源，缺失部分补画同风格简化资产。
- v1 目标不是完整豪华版，而是“四关完整可通关、关键谜题都可玩、Windows 桌面包可运行”的首个闭环版本。

## 2. 技术栈

- 前端与游戏层：Phaser 3 + TypeScript + Vite
- 桌面壳：Electron
- 打包：electron-builder
- 数据组织：TypeScript 常量数据 + JSON 场景配置
- 测试：Vitest 用于逻辑测试，Playwright 用于桌面外的 Web 交互冒烟，手工回归用于桌面壳与全流程通关

## 3. v1 功能范围

### 3.1 必做内容

1. 开场菜单与章节总进度入口。
2. 四关完整主线。
3. 线性对话系统。
4. 固定场景热点交互。
5. 四类谜题/小游戏：
   - 第一关：书房调查 + 简化棋局推理
   - 第二关：圣器对应 + 法阵摆放
   - 第三关：生火 + 寻找目标 + 村庄救援
   - 第四关：赶羊小游戏 + 结局对话
6. 基础转场、提示、失败重试。
7. 本地自动存档：至少记录当前章节、当前场景、已完成谜题。
8. Windows 可执行包与安装包。

### 3.2 v1 允许简化

1. 第一关布告由 5 份压缩为 3 份关键布告，但仍保留政治压迫与征兵信息。
2. 第一关画像自述保留画面与短文本，不做独立支线。
3. 第二关去掉花窗独立谜题，合并为法阵成功后的演出提示。
4. 第三关家书匹配从复杂多线索推理简化为清晰特征匹配。
5. 第三关救援村民固定点位，不做随机生成。
6. 第四关对话不做分支结局，统一收束到“战争结束但代价未消失”的主题表达。
7. 音频可以在首版先用少量环境音和点击音，配乐可后补。

### 3.3 延后内容

1. 成就系统。
2. 复杂分支选择。
3. 完整角色动画。
4. 多语言。
5. 设置页高级选项。

## 4. 章节实现方案

### 第一关：书房的秘密

- 场景顺序：农舍送别 -> 王宫门口 -> 宫殿接见 -> 二楼走廊 -> 书房调查
- 核心玩法：对话、阅读布告、点击线索、完成书房调查
- 必须得出的信息：国王财政紧张、战争准备、敌国勒索、主角决定从军
- 优先复用素材：
  - [Arts/第一关补充/主角家.png](Arts/第一关补充/主角家.png)
  - [Arts/第一关补充/王宫门口_画板 1.png](Arts/第一关补充/王宫门口_画板%201.png)
  - [Arts/第一关 (1)/shufang_画板 1.png](Arts/第一关%20(1)/shufang_画板%201.png)
  - [Arts/第一关 (1)/duihuakang_画板 1.png](Arts/第一关%20(1)/duihuakang_画板%201.png)
- 需要补画：国王接见场景简化版、走廊可点击画像框、书房调查弹窗图标

### 第二关：圣器的试炼

- 场景顺序：教堂 -> 藏经室 -> 圣器室 -> 法阵室 -> 高塔问答
- 核心玩法：阅读经文、对应圣器、拖放法阵、回答谜语
- 必须得出的信息：宗教叙事强化战争正当性，舞者以诱惑与象征测试主角
- 优先复用素材：
  - [Arts/第二关部分/教堂场景_画板 1.png](Arts/第二关部分/教堂场景_画板%201.png)
  - [Arts/第二关部分/舞者-01.jpg](Arts/第二关部分/舞者-01.jpg)
- 需要补画：藏经室书桌、圣器图标、法阵底图、高塔问答界面

### 第三关：战争的代价

- 场景顺序：营地生火 -> 医帐登记 -> 村庄袭掠 -> 骑士营帐 -> 战场昏迷
- 核心玩法：节奏点击、找目标、藏匿村民、阅读征税与军饷文件
- 必须得出的信息：战争后勤匮乏、抢掠被合理化、税收转嫁给民众、主角开始动摇
- 优先复用素材：
  - [Arts/第三关部分/野外_画板 1.png](Arts/第三关部分/野外_画板%201.png)
  - [Arts/第三关部分/生火_画板 1.png](Arts/第三关部分/生火_画板%201.png)
  - [Arts/第三关部分/第二幕-04.png](Arts/第三关部分/第二幕-04.png)
  - [Arts/第三关部分/触发点-06.png](Arts/第三关部分/触发点-06.png)
- 需要补画：医帐人物剪影、村庄火场、营帐文书面板、昏迷过场插图

### 第四关：战争的终结

- 场景顺序：农舍苏醒 -> 村中闲逛 -> 羊圈小游戏 -> 儿童对话 -> 停战消息
- 核心玩法：观察、赶羊、对照前文谜语的平民答案
- 必须得出的信息：战争在官方胜利后仍留下真实创伤，和平不是宣传口号而是普通人的日常愿望
- 优先复用素材：当前目录缺可直接落地 PNG，需要按既有风格补画村舍、村民、儿童与羊圈
- 需要补画：第四关主场景组、羊圈小游戏元素、结局演出背景

## 5. 缺失资产处理策略

1. 能直接复用的 PNG 优先直接接入，不重复绘制。
2. 只有局部元素、没有完整场景的素材，转为 UI 元件、调查弹窗、道具卡片或角色头像。
3. 缺失但必须存在的背景与交互面板，优先用 SVG 补画，保持扁平矢量风。
4. 补画的颜色、描边、造型要遵守 [analysis/pdf_pages/page-05.png](analysis/pdf_pages/page-05.png)、[analysis/pdf_pages/page-06.png](analysis/pdf_pages/page-06.png)、[analysis/pdf_pages/page-07.png](analysis/pdf_pages/page-07.png) 中的风格：
   - 浅米色底
   - 低饱和棕、金、蓝、红
   - 几何化人物与建筑
   - 不追求写实纹理
5. 缺失角色首版以半身像、剪影或图标代替，保证叙事信息优先落地。

## 7. 测试流程

### 7.1 测试门禁

每完成一个里程碑必须按以下顺序过门：

1. 逻辑测试
2. 交互冒烟测试
3. 章节手工通关
4. 全链路回归

任何 P0 问题未解决，不进入下一个里程碑。

### 7.2 自动化测试范围

#### P0 逻辑测试

- 场景状态机是否按顺序推进
- 谜题成功/失败判定是否正确
- 自动存档写入与恢复是否正常
- 章节完成条件是否不会漏判或误判

#### P1 交互测试

- 热点点击是否触发正确事件
- 对话框翻页、关闭、选择是否稳定
- 拖放摆放是否吸附到正确槽位
- 生火与赶羊小游戏的计时和失败判定是否正确

#### P2 构建测试

- Web 构建是否成功
- Electron 启动是否成功
- 打包产物是否可运行

### 7.3 手工测试清单

#### 章节回归

1. 第一关完整通关一次。
2. 第二关完整通关一次。
3. 第三关完整通关一次。
4. 第四关完整通关一次。

#### 全流程回归

1. 从新游戏进入，连续玩到结局。
2. 每章中途退出再启动，检查自动存档恢复。
3. 对每个谜题分别做一次失败路径验证。
4. 对关键 UI 在 1280x720 与 1920x1080 下做显示验证。

#### 桌面包验证

1. 安装。
2. 首次启动。
3. 存档写入。
4. 再次启动读取。
5. 卸载。

## 8. 当前开发决策

1. 第一版按固定场景点击叙事解谜实现，不再按横版移动原型推进。
2. 缺失美术不等待完整补齐，优先补做同风格 SVG 与简化 PNG，先保证四关完整可玩。
3. 所有新增代码与数据都围绕“四关通关闭环”服务，不提前扩展成开放式框架。
